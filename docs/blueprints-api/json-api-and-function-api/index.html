JSON API and Function API

Blueprints are defined in JSON format, but the underlying implementation uses JavaScript functions to execute the steps. While JSON is the most convenient way of interacting with Blueprints, you can also use the underlying functions directly.

JSON is merely a wrapper around the functions. Whether you use the JSON steps or the exported functions, you'll have to provide the same parameters (except for the step name):

ActivatePluginStep​

Parameters

  • pluginPath (string)

ActivateThemeStep​

Parameters

  • themeFolderName (string)

ApplyWordPressPatchesStep​

Parameters

  • addPhpInfo (boolean)
  • disableSiteHealth (boolean)
  • disableWpNewBlogNotification (boolean)
  • patchSiteUrl (boolean)
  • patchSqlitePlugin (boolean)
  • siteUrl (string)
  • wordpressPath (string)

CpStep​

Copies a file from one path to another.

Parameters

  • fromPath (string)
  • toPath (string)

Example

  • Blueprint API
  • Function API
{
    "landingPage": "/index2.php",
    "steps": [
        {
            "step": "cp",
            "fromPath": "/wordpress/index.php",
            "toPath": "/wordpress/index2.php"
        }
    ]
}
import { cp } from '@wp-playground/blueprints';

cp(
    playground,
    {
        "fromPath": "/wordpress/index.php",
        "toPath": "/wordpress/index2.php"
    },
    progress
)

InstallPluginStep​

Installs a WordPress plugin in the Playground. Technically, it uses the same plugin upload form as a WordPress user would, and then activates the plugin if needed.

Parameters

  • options (InstallPluginOptions) – Optional installation options.
  • pluginZipFile (ResourceType) – The plugin zip file to install.

Example

  • Blueprint API
  • Function API
{
    "landingPage": "/wp-admin/plugins.php",
    "steps": [
        {
            "step": "login"
        },
        {
            "step": "installPlugin",
            "pluginZipFile": {
                "resource": "wordpress.org/plugins",
                "slug": "gutenberg"
            },
            "options": {
                "activate": true
            }
        }
    ]
}
import { installPlugin } from '@wp-playground/blueprints';

installPlugin(
    playground,
    {
        "pluginZipFile": await fetchMyFile(),
        "options": {
            "activate": true
        }
    },
    progress
)

LoginStep​

Logs in to the Playground. Under the hood, this function submits the wp-login.php form just like a user would.

Parameters

  • password (string) – The password to log in with. Defaults to 'password'.

Example

  • Blueprint API
  • Function API

        {
            "step": "login",
            "username": "admin",
            "password": "password"
        }
    ]
}
import { login } from '@wp-playground/blueprints';

login(
    playground,

You can use Blueprints both with the web and the node.js version of WordPress Playground.

Differencese between JSON and Function APIs​

There are two main differences between the JSON and Function APIs:

  1. Blueprints handle the progress bar and error reporting for you. The function API requires you to handle these yourself.
  2. The function API requires importing the API client library while Blueprints may be just pasted into the URL fragment.
